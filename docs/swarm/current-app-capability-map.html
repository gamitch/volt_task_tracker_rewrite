1. Identity model — the foundational difference

The current app has no authentication at all. It is a public URL; anyone with the link sees every surface, including the coach view. Students identify via /me — a "Who are you?" name picker with honor-system That's me buttons.

Two consequences for the rewrite. (1) Students — especially FLL-age — do not manage accounts. Any attendance workflow that requires a student login will not survive contact with reality; coach-managed and low-friction paths must carry the weight. (2) The current app world-publishes children's first names, teams, and hour totals, including a public leaderboard — VOLT's auth, RLS, and leaderboard anonymization close a genuine safety exposure, not a theoretical one.

2. Coach dashboard

Far richer than a stat strip. Top to bottom: KPI cards (hours with per-category breakdown, active students per team, events logged, % toward season goal); a gear-opened inline default season-goal editor (90h, per-student override in Roster); six secondary stat tiles (avg hours/active student, students at goal, session days logged, attendance rate, upcoming 30-day commitment, busiest day); Next up; a signup/drop activity feed with Self badges when students acted from /me; hours-by-team bars; a top-10 leaderboard (full first names — see the privacy note above); per-student goal projection (stacked confirmed + planned bars against the goal line, with "156% · On track" / "18h short" annotations and a Below-goal filter); and top events by student hours.

3. Events list — density done right

Category filter (All/Outreach/Meetings/Competitions), Upcoming + Past. Each row answers everything at a glance: date range with weekday-occurrence chips ("MON (18) · THU (18)"), title + category badge, location, planned/logged hours plus "offered" hours, expected/attended counts, people reached, inline EDIT and × buttons, and a "+" expander that opens per-session detail (date, time, hours, RSVP names, reached) in place without leaving the list.

4. Upcoming Events — action cards

A working view for the next sessions: Edit / RSVP, Mark day complete, Mark whole event complete on every card, expected counts, planned hours, and RSVP'd names grouped under team chips.

5. New event form

A single well-spaced page: name, location, category buttons, times with computed duration badge; three schedule modes — Single/multi-day, Recurring (weekday-pattern series that generates e.g. 18 Mondays + 18 Thursdays), Custom dates; per-day people-reached inputs; notes; adult volunteers; and an expected-attendees roster checklist grouped by team with All/Clear shortcuts. Future-dated events record checked students as planned RSVPs; hours only count toward goals on Mark complete.

6. Edit event dialog — the workflow heart

Name/category/notes/adult volunteers/location with Save details; a running "48h total" badge; Mark complete in-dialog; + Add day / Remove day; and per day, side by side: date, start/end, hours, people reached, and an attendance checklist with a per-student hours field — the coach checks who attended and adjusts each student's hours individually. This one dialog is most of what the app is for.

7. Student self view (/me)

After the picker: team + My hours (e.g. 48 across 9 events, 76h planned), then "Check off the events you volunteered at" — a card grid of all events. Tapping a card self-reports attendance with default hours; multi-day events open a day picker ("6 of 6 days · 19.50h") with shortcuts like "Upcoming only". Self-reported actions appear in the coach dashboard feed with a Self badge. This is a retroactive path — VOLT today only has forward-looking RSVP and live QR check-in.

8. Data export

Two buttons: events CSV and roster CSV. (VOLT's Reports pages already have CSV export machinery, so parity here is small.)

Roster tab (not captured — tab switch no-ops in a headless browser): student list with team assignment, active flag, per-student goal override; add-student via header icon.

10. Ranked gaps in VOLT (from live testing + this survey)

  1. Coach-managed attendance with per-student hours — pure frontend work — T114 (PRD v2) proved the staff-write policies have existed since v1; everything else stacks on it.
  2. Expected-attendees at creation (planned RSVPs from a roster checklist).
  3. Dense event rows + expand-in-place + inline Edit/×.
  4. Dashboard analytics parity: goal projection, activity feed, stat tiles, hours-by-team, top events.
  5. "Mark whole event complete" bulk action.
  6. Retroactive attendance path young students can operate (design decision: authenticated self-report vs. coach-only backfill).
  7. Persistent KPI strip across coach pages.
  8. Create-form spacing (page-like layout vs. the current cramped dialog).
