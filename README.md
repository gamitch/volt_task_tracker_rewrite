# Claude Code Multi-Agent Swarm Template

A reusable project starter for running structured multi-agent development workflows with Claude Code. Workers implement, checkers verify, and the boss decides — nothing ships until a checker signs off.

## Architecture

```
Boss Architect (fable)
│  Defines the constitution and makes all architectural decisions
│
├── Foreman Planner (sonnet)
│   Converts the ledger into compact per-task packets
│
├── Worker Implementer (sonnet)
│   Implements one task at a time; cannot self-certify
│
├── Checker Reviewer (opus)      ← reviews against acceptance criteria
├── Checker Tests (haiku)        ← runs lint, typecheck, tests, build
├── Checker Accessibility (sonnet) ← a11y, contrast, focus, dark mode
├── Checker Content (sonnet)     ← facts, quotes, protected text fidelity
│
└── Boss Arbiter (fable)
    Resolves worker/checker disputes after 3 failed attempts
```

Key invariants:
- A task is not complete because the worker says so. Only a checker can pass it.
- Workers receive compact task packets — never the full ledger or log files.
- Workers cannot edit `.claude/`, `docs/swarm/`, or any workflow files.
- After 3 failed attempts, the foreman escalates to boss-arbiter.

## Quick Start

### 1. Use this template

Click **Use this template** on GitHub, or clone directly:

```bash
git clone https://github.com/YOUR_USERNAME/claude-swarm-template.git my-project
cd my-project
```

### 2. Fill in the project brief

Edit `docs/swarm/project-brief.md` with a description of what you're building.

### 3. Open Claude Code

```bash
claude
```

### 4. Kick off planning

Paste the starter prompt from `SWARM-QUICKSTART.md`, or run:

```
/swarm-plan
```

This invokes the boss-architect to write the constitution and task ledger. Review both before proceeding.

### 5. Run tasks

```
/swarm-run T001
```

Or let the foreman pick the next unblocked task:

```
/swarm-run
```

### 6. Final acceptance

```
/swarm-check
```

## File Structure

```
.claude/
  agents/
    boss-architect.md        # Defines constitution, architectural decisions
    boss-arbiter.md          # Resolves disputes
    foreman-planner.md       # Task packets and state compression
    worker-implementer.md    # Implements one task at a time
    checker-reviewer.md      # Reviews against acceptance criteria
    checker-tests.md         # Runs lint, typecheck, tests
    checker-accessibility.md # a11y and contrast
    checker-content.md       # Facts and protected text
  skills/
    swarm-plan/SKILL.md      # /swarm-plan slash command
    swarm-run/SKILL.md       # /swarm-run slash command
    swarm-check/SKILL.md     # /swarm-check slash command
  settings.json              # Hook config
docs/swarm/
  project-brief.md           # Fill this in first
  constitution.md            # Generated by boss-architect
  task-ledger.md             # Generated by boss-architect
  state-summary.md           # Updated by foreman
  dispute-log.md             # Escalation history
  verification-log.md        # Checker sign-offs
  active/                    # In-flight task packets
SWARM-QUICKSTART.md          # Full reference guide
```

## Severity Levels

| Level | Meaning | Task outcome |
|---|---|---|
| BLOCKER | Build failure, security, data loss, a11y, or forbidden file edit | FAIL — must fix |
| MAJOR | Important functional or architectural issue | FAIL — unless boss approves deferral |
| MINOR | Non-blocking improvement | PASS — create follow-up task |
| NIT | Cosmetic preference | PASS — log only |

## Customizing for Your Project

1. **Add domain-specific checkers** — copy an existing checker agent and specialize it (e.g., `checker-performance.md` for Lighthouse budgets).
2. **Swap models** — edit the `model:` frontmatter in any agent file to change cost/quality tradeoffs.
3. **Add project conventions to the constitution** — the boss-architect will enforce them across all tasks.
4. **Extend the skills** — add slash commands for your own project workflows.

## Requirements

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) (latest)
- An Anthropic API key with access to opus, sonnet, haiku, and fable models
